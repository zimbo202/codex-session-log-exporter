# Codex Session Log Exporter

This skill copies the JSONL log for the current Codex session into the active project directory and creates a readable Markdown transcript next to it.

## When to use

Use this skill when the user asks for any of the following:

- "copy current Codex session log to this repo/project"
- "保存当前 session 日志"
- "导出当前 Codex 会话"
- "把当前 session 的 log 放到项目路径"
- "find the log file for this Codex session"

Do not use this skill for ChatGPT web conversations or non-Codex agent sessions.

## Main rule

Do not assume the newest `rollout-*.jsonl` file is always the current session.

Codex session logs are usually under:

```text
$CODEX_HOME/sessions/YYYY/MM/DD/
~/.codex/sessions/YYYY/MM/DD/
```

The script must identify the best candidate by combining multiple signals:

1. Explicit session id if available from environment variables.
2. Whether the JSONL file content mentions the current project path, git root, working directory, or branch.
3. File modification recency.
4. Whether the file still appears active.
5. Whether filename/session metadata matches a UUID-like session/thread id.
6. Confidence threshold and manual review when ambiguous.

## Standard action

From the project root, run:

```bash
python3 .agents/skills/codex-session-log-exporter/scripts/copy_codex_session_log.py --output-dir .
```

If the skill is installed globally instead of in the repo, run the script from its installed path.

Recommended output location inside the project:

```text
.codex-session-logs/
```

The script creates:

```text
.codex-session-logs/
  codex-session-YYYYMMDD-HHMMSS-<session-id-or-hash>.jsonl
  codex-session-YYYYMMDD-HHMMSS-<session-id-or-hash>.md
  codex-session-YYYYMMDD-HHMMSS-<session-id-or-hash>.meta.json
```

The Markdown transcript is generated by the script after the JSONL copy. It renders conversation messages, tool calls, and tool results in a readable GitHub-flavored Markdown format. Use `--no-markdown` only when the user asks for the raw JSONL without a transcript.

## If the script reports low confidence

Do not silently copy a random file. Instead:

1. Show the top candidate files and their scores.
2. Ask the user to choose a candidate, or rerun with `--choose`.
3. If the user insists on best effort, use `--allow-low-confidence`.

## Security rules

Session logs can contain prompts, file contents, command output, secrets, tokens, credentials, and proprietary code.

Before committing exported logs:

- remind the user to review the log;
- recommend adding `.codex-session-logs/` to `.gitignore`;
- do not upload or transmit logs externally unless the user explicitly asks.

## Implementation notes

The deterministic operation is delegated to:

```text
scripts/copy_codex_session_log.py
```

Read `references/log-detection.md` for the detection strategy and failure modes.

The JSONL-to-Markdown converter is implemented in the same script. Its parsing approach is based on public Python Codex transcript converters such as `cc2md` and `codex-transcripts`, but it has no third-party runtime dependencies.
